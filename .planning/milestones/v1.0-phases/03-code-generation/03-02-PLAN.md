    Create src/maxpat/code_validation.py with three validators:

    1. `validate_genexpr(code: str, db: ObjectDatabase | None = None) -> list[ValidationResult]`:
       - Use ValidationResult with layer="code" for all results
       - Check 1: Balanced braces -- count { and }, report error if unbalanced
       - Check 2: Semicolons -- split code by newlines, for lines containing statements (not comments, not empty, not just { or }), check that multi-statement code has semicolons. A statement is any line with an assignment (=), function call, or declaration. Warn if a non-last statement line lacks a semicolon.
       - Check 3: in/out declarations -- extract in\d+ and out\d+ references. Info-level report of detected I/O count.
       - Check 4: Param validation -- find `Param name(...)` patterns, check they have at least a default value. Warn if min/max missing (user decision: always include min/max/default).
       - Check 5: Operator validation -- extract function-call-style tokens (word followed by `(`) from code. Skip known keywords (if, else, for, while, Param, History, Buffer, Data, in, out), skip C operators. For remaining, check against gen/objects.json via ObjectDatabase (load gen domain). Report error for unknown operators.
       - If db is None, create ObjectDatabase() for gen domain lookup.

    2. `validate_js(code: str) -> list[ValidationResult]`:
       - Check 1: inlets declaration -- regex for `inlets\s*=\s*(\d+)`. Error if missing.
       - Check 2: outlets declaration -- regex for `outlets\s*=\s*(\d+)`. Error if missing.
       - Check 3: Handler presence -- check for at least one of: `function bang()`, `function msg_int(`, `function msg_float(`, `function list(`, `function anything(`. Warn if none found.
       - Check 4: outlet() index bounds -- find all `outlet\(\s*(\d+)` calls, extract index. Compare against declared outlets count. Error if index >= outlets.

    3. `validate_n4m(code: str) -> list[ValidationResult]`:
       - Check 1: require('max-api') -- regex for `require\s*\(\s*['"]max-api['"]\s*\)`. Error if missing.
       - Check 2: addHandler calls -- find all `addHandler\s*\(\s*['"]([^'"]+)['"]` patterns. Info-level report of registered handlers. Warn if name is not a string literal.
       - Check 3: maxAPI.outlet() -- check for at least one `maxAPI\.outlet\(` call. Warn if missing (script may not send data back to MAX).

    4. `detect_js_type(code: str) -> str | None`:
       - If code contains `require('max-api')` or `require("max-api")`, return "n4m"
       - If code contains `inlets =` or `inlets=`, return "js"

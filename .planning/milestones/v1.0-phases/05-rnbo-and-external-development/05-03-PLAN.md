       - attempts: int (how many build attempts were made)

       - Handle subprocess errors gracefully (commands might not exist on non-macOS)

       - Extract file, line, column, message from standard error format (`file:line:col: error: msg`)
       - Return list of dicts with keys: file, line, column, severity, message
       - Handle both "error:" and "warning:" prefixes

    - test_build_result_failure (BuildResult with errors list)
    - test_parse_compiler_errors (parses sample clang error output)
    - test_parse_compiler_errors_empty (empty stderr returns empty list)

    Note: The actual validate_mxo tests that check `file` and `lipo` commands will be tested with mock subprocess calls since CI may not have real .mxo bundles. Create a mock .mxo bundle structure in tmp_path for structural tests.

<task type="auto" tdd="true">
  <name>Task 2: Build loop with auto-fix and Min-DevKit submodule setup</name>
  <files>src/maxpat/externals.py, tests/test_externals.py</files>
  <behavior>
    - setup_min_devkit(ext_dir) runs git submodule add --recursive for min-devkit (or checks if already present)
    - build_external(ext_dir, max_attempts=5) runs cmake configure + build loop
    - Build uses "Unix Makefiles" generator (NOT Xcode -- per Pitfall #5)
    - On compile error: parse errors, attempt auto-fix for known patterns (missing includes, missing semicolons, simple type issues), recompile
    - Tracks error signatures to detect loops (same error recurring)
    - After max_attempts or unfixable error: returns BuildResult(success=False, errors=...)
    - On success: runs validate_mxo on output, returns BuildResult(success=True, mxo_path=...)
    - Compiled .mxo placed in build/ subdirectory
  </behavior>
  <action>
    Extend `src/maxpat/externals.py` with:

    1. **setup_min_devkit(ext_dir: Path) -> bool** -- initialize Min-DevKit as git submodule:
       - Check if ext_dir / "min-devkit" / "source" / "min-api" exists (already set up)
       - If not: run `git submodule add https://github.com/Cycling74/min-devkit.git min-devkit` (Pitfall #4: must use --recursive or follow up with `git submodule update --init --recursive`)
       - Verify min-api headers exist after setup
       - Return True if successful, False if setup failed

    2. **build_external(ext_dir: Path, max_attempts: int = 5) -> BuildResult** -- cmake/make build loop:
       - Create build/ subdirectory
       - Loop up to max_attempts:
         a. Configure: `cmake -G "Unix Makefiles" ..` in build/ (NOT Xcode -- Pitfall #5)
         b. Build: `cmake --build . --config Release`
         c. If success: find .mxo in build/, run validate_mxo, return BuildResult
         d. If error: parse_compiler_errors on stderr
         e. auto_fix(errors, ext_dir): attempt to fix known patterns
         f. Track error hashes -- if same error appears twice, it is unfixable
       - After loop: return BuildResult(success=False, errors=last_errors, attempts=max_attempts)

    3. **auto_fix(errors: list[dict], ext_dir: Path) -> bool** -- attempt to fix known compiler errors:
       - Missing include: add `#include "{header}"` to source file
       - Missing semicolon: add `;` at indicated line
       - Simple type mismatch: limited heuristics for common Min-DevKit types
       - Return True if any fix was applied, False if errors are unfixable
       - IMPORTANT: limit to well-known patterns only. Escalate anything complex (per Pitfall #6)

    Extend `tests/test_externals.py` with:
    - test_build_external_no_cmake (returns BuildResult failure when cmake not configured)
    - test_auto_fix_missing_semicolon (auto_fix adds semicolon at indicated line)
    - test_auto_fix_unfixable (auto_fix returns False for unknown error types)
    - test_auto_fix_loop_detection (same error hash twice triggers early exit)
    - test_setup_min_devkit_already_present (returns True when min-devkit exists)
    - test_build_invocation_mocked (mock subprocess to verify cmake commands called correctly)

    NOTE: Full integration tests (actual cmake compilation) cannot run without min-devkit installed. Unit tests mock subprocess.run to verify correct command construction and error handling flow. Mark any integration test that requires real cmake with `@pytest.mark.skipif` if cmake is not available.

    Write tests FIRST, then implement to make them pass.

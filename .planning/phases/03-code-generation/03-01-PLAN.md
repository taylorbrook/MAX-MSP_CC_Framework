    b) `build_genexpr(params: list[dict], code_body: str, num_inputs: int = 1, num_outputs: int = 1) -> str`:
       - Builds a complete GenExpr code string following the user's locked style decisions:
         - Section headers with `// === SECTION_NAME ===` format
         - Full Param range specs: `Param name(default, min=min_val, max=max_val);`
         - Descriptive variable names in code_body
       - params is list of dicts: [{"name": "freq", "default": 440, "min": 20, "max": 20000}, ...]
       - code_body is the DSP logic section (raw GenExpr code between params and outputs)
       - Returns formatted string with PARAMETERS section, then code_body, then output assignments if not already in code_body
       - Note: code_body may already contain `out1 = ...` lines -- in that case, don't add redundant output section

    c) `generate_gendsp(code: str, num_inputs: int | None = None, num_outputs: int | None = None) -> dict`:

         - Top-level "patcher" key using copy of DEFAULT_PATCHER_PROPS

         - Add "in 1", "in 2", ... boxes (maxclass: "newobj", text: "in 1") positioned at top
         - Add codebox box (maxclass: "codebox", code: code, fontname: "Arial", fontsize: 12.0)
         - Set codebox numinlets = num_inputs, numoutlets = num_outputs, outlettype = [""] * num_outputs
         - Add "out 1", "out 2", ... boxes (maxclass: "newobj", text: "out 1") positioned at bottom
         - Add patchlines: each in -> codebox inlet, each codebox outlet -> out
         - in objects: numinlets=0, numoutlets=1, outlettype=[""]
         - out objects: numinlets=1, numoutlets=0, outlettype=[]
       - Codebox patching_rect: [50.0, 80.0, 400.0, 200.0] (from research)
       - in objects positioned at y=20.0, out objects at y=320.0, spaced horizontally by 80px starting at x=50.0

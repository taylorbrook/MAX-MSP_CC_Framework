  <files>src/maxpat/rnbo.py, tests/test_rnbo.py</files>
  <behavior>
    - add_rnbo(patcher, objects, params, target, audio_ins, audio_outs) creates rnbo~ box with inner RNBO patcher
    - Inner patcher contains: in~ objects for audio inputs, out~ for outputs, param objects for parameters, inport/outport for messages, user-specified objects with connections
    - Parent rnbo~ box uses Box.__new__ pattern (not DB lookup) with variable I/O (audio ins + 1 message, audio outs + 1 message)
    - Full wrapper in parent .maxpat: adc~ -> rnbo~ -> dac~ connections, param exposure
    - Validates all objects are RNBO-compatible before building (raises ValueError if not)
    - Returns (parent_box, inner_patcher) tuple matching add_gen() return pattern
    - generate_rnbo_wrapper(patcher, params, target, audio_ins, audio_outs) creates the full ready-to-export .maxpat with dac~/adc~ connections
  </behavior>
  <action>
    Extend `src/maxpat/rnbo.py` with:

    1. **add_rnbo() function** (standalone function that takes a Patcher, following the same pattern as Patcher.add_gen but as a module-level function to avoid modifying patcher.py):
       - Validate all object names against RNBODatabase.is_rnbo_compatible() -- raise ValueError for any non-RNBO object
       - Create inner RNBO patcher with proper bgcolor and rect
       - Add in~ objects (1 through audio_ins) using Box.__new__ pattern
       - Add out~ objects (1 through audio_outs) using Box.__new__
       - Add param objects for each params entry: `param @name {name} @min {min} @max {max} @initial {default}` using Box.__new__
       - Add inport/outport objects for message I/O
       - Add user-specified objects and their connections to inner patcher
       - Create parent rnbo~ box via Box.__new__: numinlets = audio_ins + 1, numoutlets = audio_outs + 1, outlettype = ["signal"] * audio_outs + [""]
       - Set parent_box._inner_patcher = inner patcher
       - Append parent_box to patcher.boxes
       - Return (parent_box, inner_patcher)

    2. **generate_rnbo_wrapper(params, target, audio_ins, audio_outs)** -- builds a complete Patcher that is ready to open in MAX and export:
       - Creates top-level Patcher

       - Adds adc~ (audio_ins channels) connected to rnbo~ signal inlets
       - Adds dac~ (audio_outs channels) connected from rnbo~ signal outlets
       - Adds number boxes or live.dial for exposed params (optional, per discretion)
       - Returns the Patcher (caller can use generate_patch() or write_patch() on it)

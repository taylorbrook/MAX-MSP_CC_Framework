<objective>
Build the analysis engine that compares parser-extracted help patch data against the object database across 5 dimensions: outlet types, I/O counts, box widths, argument formats, and connection patterns. Produces per-object findings with confidence scores.

Purpose: This is the comparison brain of the audit -- it takes raw parsed data and identifies every discrepancy between what the help patches show and what the DB records. The confidence scoring lets downstream consumers prioritize high-confidence corrections.

Output: `src/maxpat/audit/analyzer.py` module with AuditAnalyzer class and ObjectFindings dataclass, plus comprehensive tests.
</objective>

  2. Define helper functions at module level:
     - `SIGNAL_TYPES = {"signal", "multichannelsignal"}`
     - `classify_outlet_type(help_type: str) -> tuple[bool, str]`: map help patch outlettype strings to DB signal/control classification. "signal" -> (True, "signal"), "multichannelsignal" -> (True, "multichannelsignal"), everything else -> (False, "")
     - `compute_confidence(instances: list, key_fn: callable) -> tuple[str, float, any]`: aggregate instances using key_fn, count via Counter, return (level, ratio, consensus_value). Thresholds: >= 1.0 HIGH, >= 0.75 MEDIUM, >= 0.5 LOW, else CONFLICT. Empty list returns ("NONE", 0.0, None).

  3. Define `ObjectFindings` dataclass:
     - object_name: str
     - instance_count: int
     - outlet_type_finding: dict | None (keys: db_types, help_types, confidence, agreement, discrepancy_type)
     - io_count_finding: dict | None (keys: db_inlets, db_outlets, help_inlets, help_outlets, confidence, variable_io)
     - width_finding: dict | None (keys: widths, median_width, min_width, max_width, instance_count)
     - argument_finding: dict | None (keys: patterns with counts)
     - connection_finding: dict | None (keys: outlet_connections as dict of outlet_idx -> list of (target_obj, target_inlet, count))

  4. Define `AuditAnalyzer` class:
     - `__init__(self, db: ObjectDatabase)`: store DB reference
     - `analyze_outlet_types(self, name: str, instances: list[BoxInstance]) -> dict | None`:
       (a) Get DB outlet types via db.get_outlet_types(name)
       (b) Collect outlettype arrays from all instances, classify each to signal/control
       (c) Compute confidence on the classified types
       (d) Compare consensus against DB types
       (e) Return finding dict with discrepancy details, or None if types match
       (f) Do NOT flag int/float variation as signal/control discrepancy
     - `analyze_io_counts(self, name: str, instances: list[BoxInstance]) -> dict | None`:
       (a) Look up object, check variable_io flag
       (b) For variable_io objects: parse args from each instance text via parse_object_text, compute expected counts per instance via db.compute_io_counts(name, args), compare
       (c) For fixed objects: compare instance numinlets/numoutlets against DB defaults
       (d) Compute confidence across instances
       (e) Return finding dict with discrepancy details, or None if counts match

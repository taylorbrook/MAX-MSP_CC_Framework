    - Test: Report entries include confidence level and agreement ratio for each finding
    - Test: Empty I/O tracker identifies objects with empty inlets array in DB
    - Test: Empty I/O tracker identifies objects with empty outlets array in DB
    - Test: Empty I/O tracker marks objects that have help patch data available as "has_help_data: true"
    - Test: Empty I/O tracker marks objects without help patch data as "has_help_data: false"
    - Test: Report can be filtered by dimension (--outlets-only shows only outlet type findings)
  </behavior>
  <action>
  1. Create `src/maxpat/audit/reporter.py` with `AuditReporter` class:
     - `__init__(self, db: ObjectDatabase)`: store DB reference

     - `generate_report(self, findings: dict[str, ObjectFindings], parse_stats: dict) -> dict`:
       (a) Build report dict with `_audit_date`, `_audit_stats` (from parse_stats), `summary` section
       (b) For each object with findings, create an entry containing only non-None findings
       (c) Include confidence level and agreement ratio in each finding entry
       (d) Sort objects alphabetically
       (e) Compute summary: total discrepancies by confidence (HIGH, MEDIUM, LOW, CONFLICT), by dimension (outlet_types, io_counts, widths, args, connections)
       (f) Return complete report dict

     - `filter_report(self, report: dict, dimension: str) -> dict`:
       (a) Filter report entries to only show the specified dimension
       (b) Supported dimensions: "outlet_types", "io_counts", "widths", "arguments", "connections"
       (c) Remove objects that have no finding for the requested dimension
       (d) Return filtered report dict

     - `find_empty_io_objects(self, findings: dict[str, ObjectFindings]) -> dict`:
       (a) Iterate over all objects in DB (access via db._objects)
       (b) Identify objects with empty inlets or empty outlets arrays
       (c) For each empty-I/O object, check if findings dict has data for it
       (d) Return dict keyed by object name with: empty_inlets (bool), empty_outlets (bool), has_help_data (bool), proposed_inlets (list|None), proposed_outlets (list|None), domain (str)
       (e) Include summary: total_empty_inlet, total_empty_outlet, total_unique, covered_by_help, not_covered

     - `write_report(self, report: dict, output_path: Path) -> None`:
       (a) Write report dict as pretty-printed JSON to output_path
       (b) Ensure output directory exists (mkdir -p)

  2. Write tests in `tests/test_audit_reporter.py`:
     - Create mock ObjectFindings with various combinations of findings
     - Test report structure and summary stats
     - Test filtering by dimension
     - Test empty I/O tracker with mock DB containing objects with empty inlets/outlets

  <name>Task 2: Implement proposed override generation with manual entry protection</name>
  <files>src/maxpat/audit/overrides.py, tests/test_audit_reporter.py</files>
  <behavior>
    - Test: Generated proposed overrides match the established overrides.json "objects" dict format (inlets/outlets arrays with id, type, signal, digest keys)
    - Test: Objects already in existing overrides.json are flagged as "CONFLICT_WITH_MANUAL" and NOT included in proposed overrides
    - Test: Conflicting entries include both audit-proposed and existing-manual values for human review
    - Test: Only HIGH and MEDIUM confidence findings generate proposed overrides (LOW and CONFLICT are excluded)
    - Test: Proposed overrides include _audit metadata (confidence, instances, agreement, source_files)
    - Test: Proposed overrides for outlet types correctly construct outlet array entries from help patch outlettype strings
    - Test: Empty I/O objects with help patch data generate proposed I/O entries in proposed overrides
  </behavior>
  <action>
  1. Create `src/maxpat/audit/overrides.py` with `OverrideGenerator` class:
     - `__init__(self, db: ObjectDatabase, overrides_path: Path | None = None)`:
       (a) Store DB reference
       (b) Load existing overrides.json from overrides_path (default: project_root/.claude/max-objects/overrides.json)
       (c) Extract existing manually-corrected object names from overrides["objects"]

     - `generate_proposed_overrides(self, findings: dict[str, ObjectFindings], empty_io_data: dict | None = None) -> dict`:
       (a) Initialize proposed dict with `_comment`, `_audit_date`, `_audit_stats`, `objects` dict, `conflicts` dict
       (b) For each object with outlet_type_finding:
           - If confidence is HIGH or MEDIUM AND there is a discrepancy:
           - If object name exists in existing manual overrides: add to `conflicts` dict with both values, DO NOT add to `objects`
           - Otherwise: build outlet array from help patch consensus outlettype, add to `objects` with `_audit` metadata
       (c) For each object with io_count_finding (HIGH/MEDIUM confidence):
           - Same conflict check logic
           - Build inlet/outlet arrays from help patch consensus counts
       (d) For empty_io_data objects where has_help_data is True:
           - If object not in existing manual overrides: propose I/O entries from help patch data
       (e) Return complete proposed overrides dict

     - `_build_outlet_array(self, outlettype: list[str]) -> list[dict]`:
       (a) For each type string in outlettype: create dict with id, type, signal (True if in SIGNAL_TYPES), digest ("Signal output" or "Control output")
       (b) Return list of outlet dicts matching overrides.json format

     - `_build_inlet_array(self, count: int) -> list[dict]`:
       (a) Create count inlet dicts with id, type="", signal=False, digest=""
       (b) Return list of inlet dicts

     - `write_proposed_overrides(self, proposed: dict, output_path: Path) -> None`:
       (a) Write as pretty-printed JSON
       (b) Ensure output directory exists

     - Test _audit metadata presence
     - Test outlet array construction from help patch types
     - Test empty I/O object proposals

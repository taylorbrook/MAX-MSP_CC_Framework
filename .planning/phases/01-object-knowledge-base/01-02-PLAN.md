**5. merge_sources.py** -- Script that enriches base extraction:

```
python3 .claude/scripts/merge_sources.py
```

The script:
a. Loads all domain objects.json files from `.claude/max-objects/*/`
b. Loads overrides.json and deep-merges overrides onto matching objects (override fields replace extracted fields)
c. Loads the RNBO objects from `.claude/max-objects/rnbo/objects.json` -- builds a set of RNBO object names. For each object in all other domains, sets `rnbo_compatible: true` if the object name (without mc. prefix) appears in the RNBO name set, otherwise `rnbo_compatible: false`. Note: RNBO names in refpages may be prefixed with "rnbo_" in the filename -- strip that prefix when building the set.
d. Applies version tagging: reads `version_map` from overrides.json. For objects matching MAX 9 prefixes (array.*, string.*, abl.*), sets `min_version: 9`. For MC prefix objects, sets `min_version: 8`. All others default to `min_version: 8` if no min_version already set.
e. Applies `variable_io_rules` from overrides.json to matching objects -- sets `variable_io: true` and copies the `io_rule` dict.
f. Writes enriched objects back to their domain JSON files (in-place update).
g. Prints summary: objects enriched, RNBO flags set, version tags applied, overrides applied.

CLI: `python3 .claude/scripts/merge_sources.py [--overrides path] [--dry-run]`

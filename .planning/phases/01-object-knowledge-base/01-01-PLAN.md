**Extraction script (.claude/scripts/extract_objects.py):**

Create a Python script using only stdlib (xml.etree.ElementTree, json, pathlib, argparse, collections) that:

1. Discovers all .maxref.xml files across:
   - `/Applications/Max.app/Contents/Resources/C74/docs/refpages/max-ref/` (Max domain)
   - `/Applications/Max.app/Contents/Resources/C74/docs/refpages/msp-ref/` (MSP domain)
   - `/Applications/Max.app/Contents/Resources/C74/docs/refpages/jit-ref/` (Jitter domain)
   - `/Applications/Max.app/Contents/Resources/C74/docs/refpages/m4l-ref/` (M4L domain)
   - `/Applications/Max.app/Contents/Resources/C74/packages/Gen/docs/refpages/` (Gen domain -- uses different XML schema)
   - `/Applications/Max.app/Contents/Resources/C74/packages/RNBO/docs/refpages/` (RNBO -- extracted but stored separately for Plan 02 cross-reference)
   - `/Applications/Max.app/Contents/Resources/C74/packages/*/docs/refpages/` (Other packages)

2. Parses each XML file using two parser branches:
   - **Standard parser** for core refpages: Extracts from `<c74object>` root: name, module, category, kind, digest, description, inletlist, outletlist, objarglist, methodlist, attributelist, seealsolist, metadatalist tags.
   - **Gen~ parser** for Gen package: Handles `<geninletlist>`, `<constructorlist>`, `kind="genobject_dsp"/"genobject_jit"`. Gen~ outlets default to 1 signal outlet when `<genoutletlist>` is absent. Inlets come from `<geninletlist>/<geninlet>` with `@name` and `@type`.

3. Normalizes during extraction:
   - **Module normalization:** Map {"max", "Max", "core"} -> "max", {"msp", "MSP"} -> "msp", {"jit"} -> "jit", {"m4l"} -> "m4l". Empty module: infer from file path (msp-ref/ -> msp, max-ref/ -> max, jit-ref/ -> jit).
   - **Domain assignment:** "max" -> "Max", "msp" -> "MSP" (but mc.* prefix objects -> "MC"), "jit" -> "Jitter", "m4l" -> "M4L", gen-dsp/gen-jit/gen-common -> "Gen", packages -> "Packages".
   - **Inlet type normalization:** Apply the INLET_TYPE_MAP from research (591 objects with INLET_TYPE, 520 with OUTLET_TYPE). Map all variants to normalized set: signal, signal/float, signal/int, signal/message, mc_signal, matrix, control, int, float, bang, list, anything, dict, etc.
   - **Signal boolean:** Set `signal: true` for signal, signal/float, signal/int, signal/message, signal/list, signal/mc, mc_signal types.
   - **Hot/cold inference:** Inlet 0 = hot for control objects. All signal inlets = hot for MSP objects. Others = cold. Store as `hot: true/false` per inlet.
   - **maxclass assignment:** Set `maxclass` to object name (this is the standard convention in MAX -- the maxclass equals the object name for most objects). For Gen~ objects, set maxclass to "gen~" (they live inside gen~ patchers).

4. Builds normalized object entries following the schema from research:
   ```json
   {
     "name": "cycle~",
     "maxclass": "cycle~",
     "module": "msp",
     "domain": "MSP",
     "category": "MSP Synthesis",
     "digest": "Sinusoidal oscillator",
     "description": "...",
     "inlets": [{"id": 0, "type": "signal/float", "signal": true, "digest": "Frequency", "hot": true}, ...],
     "outlets": [{"id": 0, "type": "signal", "signal": true, "digest": "Output"}],
     "arguments": [{"name": "frequency", "type": "number", "optional": true, "digest": "..."}],
     "messages": ["float", "signal", ...],
     "attributes": {"frequency": {"type": "float", "get": true, "set": true}},
     "seealso": ["buffer~", "phasor~"],
     "tags": ["MSP", "oscillator"],
     "min_version": 8,
     "verified": true,
     "variable_io": false
   }
   ```

5. Detects variable I/O objects by name matching against known list: pack, unpack, route, routepass, select (sel), trigger (t), gate, switch, funnel, spray, matrix~, selector~, router, join, unjoin, combine, cycle (Max, not MSP cycle~). For these, set `variable_io: true` and add `io_rule` with description, inlet_count rule, outlet_count rule, and default counts. The specific rules per object:
   - `trigger/t`: fixed 1 inlet, outlets = arg count (default 2)
   - `pack`: inlets = arg count (default 2), fixed 1 outlet
   - `unpack`: fixed 1 inlet, outlets = arg count (default 2)
   - `route`: inlets = 1, outlets = arg count + 1 (default 3)
   - `select/sel`: inlets = 1, outlets = arg count + 1 (default 2)
   - `gate`: inlets = 2, outlets = first arg (default 1)
   - `switch`: inlets = first arg + 1 (default 2), outlets = 1
   - `spray`: inlets = 1, outlets = first arg (default 2)
   - `funnel`: inlets = first arg (default 2), outlets = 1
   - `matrix~`: inlets = first arg (default 1), outlets = second arg (default 1)
   - `selector~`: inlets = first arg + 1 (default 2), outlets = 1

6. Writes output:
   - One `objects.json` per domain directory under `.claude/max-objects/` -- each file is a JSON object keyed by object name with the object dict as value
   - MC objects (name starts with `mc.`) go to `.claude/max-objects/mc/objects.json` even though they're in msp-ref
   - RNBO objects extracted and stored temporarily for Plan 02 cross-referencing (write to `.claude/max-objects/rnbo/objects.json` as RNBO-specific entries)
   - `extraction-log.json` with: total files found, total objects extracted, per-domain counts, objects with INLET_TYPE fallback, objects with variable_io, objects with empty inlets/outlets, extraction timestamp, MAX installation path

7. CLI interface via argparse:
   - `python3 .claude/scripts/extract_objects.py` (default: extract all)
   - `python3 .claude/scripts/extract_objects.py --max-path /path/to/Max.app` (custom MAX path)
   - `python3 .claude/scripts/extract_objects.py --domain msp` (extract single domain)
   - `python3 .claude/scripts/extract_objects.py --dry-run` (count files, don't extract)

Handle XML parsing errors gracefully -- log the file path and error, continue extraction, report in extraction-log.json.

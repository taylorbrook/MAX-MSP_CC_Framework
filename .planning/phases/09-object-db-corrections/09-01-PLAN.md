Create `src/maxpat/audit/merger.py` with an `OverrideMerger` class:

```python
class OverrideMerger:
    def __init__(self, proposed_path: Path, overrides_path: Path, db: ObjectDatabase) -> None
    def detect_conflicts(self) -> dict[str, dict]
    def merge(self, conflict_resolutions: dict | None = None) -> dict
    def write(self, merged: dict) -> None
```

Implementation details:

**`__init__`**: Load proposed-overrides.json and existing overrides.json. Store both for comparison. Store db for domain lookups.

**`detect_conflicts`**: Return the `conflicts` dict from proposed-overrides.json. Each conflict has `audit_proposed`, `existing_manual`, and `reason` keys.

**`merge(conflict_resolutions)`**:
1. Start with deep copy of existing overrides.json (preserves _comment, version_map, variable_io_rules).
2. Collect all objects to merge: proposed["objects"] (211 non-conflict entries) + resolved conflict entries.
3. For conflict_resolutions: dict maps object name to dict of `{"adopt_fields": ["inlets", ...]}` -- fields to take from audit, rest preserved from manual. Unresolved conflicts (None or missing) are skipped (not merged).
4. For each resolved conflict: deep copy manual entry, store as `_manual_original`, adopt specified fields from audit, add `_audit` metadata.
5. Merge non-conflict objects: include ALL confidence tiers (HIGH, MEDIUM, HELP_PATCH). Each retains its existing `_audit` metadata dict.
6. For the 40 objects not in any domain JSON: add `_note: "No base domain entry -- override inactive until object added to domain JSON"`.
7. Group all objects (existing manual non-conflicts + new audit entries + resolved conflicts) by domain using `_domain_header` separator keys.
8. Domain ordering: max, msp, jitter, mc, gen, m4l, rnbo, packages, other (for orphans).
9. Domain detection: use `db.lookup(name)` then read `domain` field. If None, check `module` field. If still None, domain is "other".
10. Return the complete merged dict.

**`write(merged)`**: Write as pretty-printed JSON (indent=2) with trailing newline.

CRITICAL PITFALLS TO AVOID:
- Do NOT lose version_map or variable_io_rules -- start from deep copy of existing file, only replace `objects` key.
- Do NOT replace manual digest strings with generic "Control output" -- only adopt fields explicitly listed in conflict_resolutions.
- Do NOT create duplicate entries -- use proposed-overrides.json as single source (the 43 empty-I/O overlap objects are already in proposed["objects"]).
- Existing manual entries that are NOT in the 8 conflicts must remain exactly as they appear, just re-grouped into their domain section.
- `_domain_header` keys are underscored so db_lookup.py skips them during merge.

    Create `.claude/commands/` directory and 5 command files. Each follows this format:
    ```
    ---
    name: max-{name}
    description: {one-line description}
    argument-hint: "{hint}"
    ---
    # /max-{name}
    {Detailed behavior instructions for Claude}
    ```

    **max-new.md:**
    - argument-hint: "[project-name]"
    - Behavior: Validate name (lowercase alphanumeric + hyphens). Call create_project() from src.maxpat.project. Set as active project. Start conversational kickoff: ask user to describe what they want to build, ask 3-5 clarifying questions about audio/MIDI requirements, signal flow, UI needs, target MAX version. Write answers to project's context.md.
    - Examples: `/max-new my-synth`, `/max-new step-sequencer`

    **max-discuss.md:**
    - argument-hint: "[topic?]"
    - Behavior: Load active project context.md. Engage in discussion about implementation decisions. Capture decisions and append to context.md. Update status stage to "discuss".
    - Examples: `/max-discuss filter design`, `/max-discuss`

    **max-research.md:**
    - argument-hint: "[topic?]"
    - Behavior: Load active project context.md. Research MAX-specific approaches: object choices, signal flow patterns, existing techniques. Read relevant object database domains. Write findings to project context.md research section. Update status stage to "research".
    - Examples: `/max-research granular synthesis approaches`, `/max-research`

    **max-status.md:**
    - No argument-hint needed
    - Behavior: Read active project from .active-project.json. Read project status.md. Display: project name, current stage, progress, files generated, last activity. List all projects if no active project. Show warning if active project desync detected.
    - Examples: `/max-status`

    **max-switch.md:**
    - argument-hint: "[project-name]"
    - Behavior: Validate project exists in patches/. Call set_active_project(). Display confirmation with project status summary. If name not provided, list available projects for selection.
    - Examples: `/max-switch my-synth`, `/max-switch`
  </action>

<task type="auto">
  <name>Task 2: Create generation commands (build, iterate, verify, test, memory) and validation tests</name>
  <files>.claude/commands/max-build.md, .claude/commands/max-iterate.md, .claude/commands/max-verify.md, .claude/commands/max-test.md, .claude/commands/max-memory.md, tests/test_commands.py</files>
  <action>
    **max-build.md:**
    - argument-hint: "[description]"
    - Behavior: Load active project. Invoke max-router skill with user's description + project context + relevant memory. Router dispatches to specialist agent(s). Run critic loop on output (max-critic skill). Write output to project's generated/ directory. Write-back memory (max-memory-agent). Update status stage to "build", increment progress.
    - Examples: `/max-build subtractive synth with filter envelope`, `/max-build gen~ waveshaper with drive and mix`

    **max-iterate.md:**
    - argument-hint: "[changes]"
    - Behavior: Load active project. Read existing generated files. User describes modifications. Route through max-router for the relevant domain. Apply changes to existing files (not regenerate from scratch). Run critic loop. Write-back memory. Update progress.
    - Examples: `/max-iterate add LFO to filter cutoff`, `/max-iterate change metro rate to 200ms`

    **max-verify.md:**
    - No argument-hint needed
    - Behavior: Load active project. Find all generated files. Run validation pipeline (validate_file/validate_code_file from src.maxpat). Run critic loop (review_patch from src.maxpat.critics). Display results grouped by severity. For blockers, suggest fixes. For warnings, annotate. Update status.
    - Examples: `/max-verify`

    **max-test.md:**
    - argument-hint: "[patch-name?]"
    - Behavior: Load active project. If patch name given, generate test checklist for that patch. If not, generate for all patches in generated/. Use generate_test_checklist() from src.maxpat.testing. Display checklist. After user reports results, save via save_test_results(). Update project status with test results.
    - Examples: `/max-test my-synth.maxpat`, `/max-test`

    **max-memory.md:**
    - argument-hint: "[list|view|forget] [domain?]"
    - Behavior:
      - `list`: Show all domains with entry counts for both project and global memory
      - `view [domain]`: Show all entries for domain (or all domains)
      - `forget [pattern-name] [domain]`: Delete specific memory entry
    - Uses MemoryStore from src.maxpat.memory for all operations
    - Examples: `/max-memory list`, `/max-memory view dsp`, `/max-memory forget "prefer line~ for gain" dsp`

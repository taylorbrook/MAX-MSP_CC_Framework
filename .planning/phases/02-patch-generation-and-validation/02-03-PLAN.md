  <action>
    Create `src/maxpat/validation.py`:

    `class ValidationResult`:
    - Fields: layer (str: "json", "objects", "connections", "domain"), level (str: "error", "warning", "info", "fixed"), message (str), auto_fixed (bool, default False).
    - `__repr__`: "[{layer}:{level}] {message}"

    `def validate_patch(patch, db=None) -> list[ValidationResult]`:
    - Accepts Patcher instance or dict. If Patcher, calls to_dict() to get the dict form. If db is None and patch is Patcher, uses patch.db.
    - Runs all four layers in order. If Layer 1 finds structural errors, stops early (nothing else can run).
    - Returns combined results from all layers.

    `def has_blocking_errors(results) -> bool`:
    - Returns True if any result has level=="error" and auto_fixed==False.

    **Layer 1: validate_json_structure(patch_dict) -> list[ValidationResult]:**
    - Check "patcher" key exists
    - Check "patcher" contains "boxes" (must be list)
    - Check "patcher" contains "lines" (must be list)
    - Check each box entry has "box" wrapper with "maxclass" and "id"
    - Check each line entry has "patchline" wrapper with "source" and "destination"

    **Layer 2: validate_objects_exist(patch_dict, db) -> list[ValidationResult]:**
    - For each box: extract object name from "text" field (if maxclass=="newobj", split text and take first token) or from "maxclass" (for UI objects).
    - Look up in ObjectDatabase. If not found, check if PD object -> error with suggestion. If simply unknown -> error.
    - Skip inlet, outlet, patcher, bpatcher maxclasses (these are structural, not user objects).

    **Layer 3: validate_connections(patch_dict, db) -> list[ValidationResult]:**
    - Build box lookup: id -> box dict.
    - For each patchline:
      - Resolve source box, check source outlet index < numoutlets. If out of bounds: error + auto-fix (mark for removal).
      - Resolve dest box, check dest inlet index < numinlets. If out of bounds: error + auto-fix (mark for removal).
      - Check signal type compatibility: get source outlet type from outlettype array. Get dest inlet signal type from database. If source is "signal" and dest inlet is control-only (signal: false on inlet AND inlet is not a "signal or float" type): warning + auto-fix (mark for removal). Note: per CLAUDE.md, signal/float inlets accept both signal and control -- so only flag when inlet explicitly cannot accept signal.
    - Return auto-removed connections as ValidationResult with auto_fixed=True and level="fixed".
    - Also return a mutated copy of lines with invalid connections removed (store on result or return separately).

    **Layer 4: validate_domain_rules(patch_dict, db) -> list[ValidationResult]:**
    - Build signal graph from connections (only signal-type connections).
    - **Unterminated signal chains:** Find MSP objects (name ends with ~) that have signal outlets but no downstream signal connection. Exceptions: dac~, ezdac~, send~, *~ 0. (muted), out~. Warn for each unterminated chain.
    - **Missing gain staging:** Find direct connections from oscillator objects (cycle~, saw~, rect~, tri~, noise~, pink~) to dac~/ezdac~ without passing through a gain object (*~, gain~, *~ with value < 1.0). Warn for each.
    - **Feedback loop detection:** Run DFS on signal graph to detect cycles. If a cycle is found, check if it contains tapin~/tapout~ or passes through gen~. If not, warn "signal feedback loop without delay detected."
    - **Hot/cold inlet ordering:** For non-signal connections from a single outlet to multiple inlets of the same object: check if hot inlet (index 0) receives before cold inlets. If connections in lines array have hot inlet earlier than cold inlets of the same target, warn. (This is informational -- difficult to auto-fix since it depends on execution order.)

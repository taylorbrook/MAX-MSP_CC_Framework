    - Column x-positions: first column at x=30, each subsequent column at previous_x + widest_box_in_prev_column + H_GUTTER
    - Objects within column are left-aligned (same x coordinate)
    - UI control objects (toggle, slider, dial, number, flonum) that connect to an object in the same or next column are positioned directly above their target
    - Comment objects placed above the first object in their associated column/section
    - Empty patcher (no boxes): no-op, returns without error
    - Single box (no connections): placed at default position
    - Presentation mode: when patcher has UI objects, generate presentation_rect for UI objects grouping them in a grid layout

  <action>
    Create `src/maxpat/layout.py`:

    `def apply_layout(patcher: Patcher) -> None`:
    Main entry point. Mutates box patching_rect positions in-place. Steps:
    1. Early return if no boxes.
    2. Build adjacency graph from patcher.lines (source_id -> [dest_id]).
    3. Run Kahn's algorithm (topological sort by levels) to assign boxes to columns. Use the algorithm from 02-RESEARCH.md Example 2 but adapted to work with Box objects.
    4. Handle disconnected nodes: boxes not reached by topological sort go in a final column.
    5. Within each column, sort boxes to maintain signal flow coherence: objects that connect to objects in the same column's lower positions should be above those targets.
    6. Assign x-positions: first column starts at x=30.0. Each subsequent column's x = previous column's x + max(width of all boxes in previous column) + H_GUTTER.
    7. Assign y-positions: within each column, first box at y=30.0, each subsequent box at previous_y + previous_height + V_SPACING.
    8. UI control adjustment: for UI objects (toggle, slider, dial, number, flonum) that have a connection to an object, position them directly above their first target (same x, y = target_y - ui_height - V_SPACING/2). If this creates overlap with other objects, keep the column-assigned position.
    9. Comment positioning: comments associated with a section get placed at the top of the column they logically belong to (above the first non-comment box, offset by 30px).
    10. Apply presentation mode layout if any boxes have presentation=True: arrange presentation_rect values in a grid (left-to-right, wrapping). Use presentation_rect = [20 + col*spacing, 20 + row*spacing, w, h].

    Helper functions:
    - `_build_graph(boxes, lines) -> tuple[dict, dict]`: Returns (adjacency dict, in-degree dict).
    - `_topological_columns(boxes, adj, in_degree) -> list[list[Box]]`: Kahn's algorithm returning list of columns, each a list of Box objects.
    - `_compute_column_positions(columns) -> list[float]`: Returns x-position for each column.
    - `_apply_presentation_layout(boxes) -> None`: Sets presentation_rect on boxes with presentation=True.

    - Presentation layout: UI boxes get presentation_rect values

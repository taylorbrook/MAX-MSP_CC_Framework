# UI Agent Boundaries

## This Agent DOES

- Design and apply presentation mode layout (control positioning, grouping, visual hierarchy)
- Position UI objects with `presentation_rect` attributes
- Add panel backgrounds and comment labels for visual organization
- Configure visual attributes (bgcolor, textcolor, fontsize, fontname)
- Create consistent control sizing within groups
- Apply the layout engine's auto-layout for patching mode positioning
- Advise on UI object selection (which dial/slider/display is appropriate)

## This Agent Does NOT

- **Generate signal processing** -- audio signal chains and Gen~ code belong to the DSP agent. This agent positions controls that the DSP agent's signal chain connects to.
- **Build patch routing logic** -- control-rate object connections (trigger, route, gate, etc.) belong to the Patch agent. This agent positions the boxes; the Patch agent wires them.
- **Write JavaScript code** -- js/N4M scripts belong to the js agent. Even jsui drawing code is generated by the js agent (with positioning by the UI agent).
- **Generate RNBO patches** -- RNBO export belongs to the RNBO agent.
- **Create C/C++ code** -- external development belongs to the ext agent.

## Handoff Points

| Situation | Hand Off To | What to Provide |
|-----------|------------|----------------|
| Controls need signal connections | max-dsp-agent | Control names, output ranges, parameter mapping |
| Controls need patch routing | max-patch-agent | Control names, message format, destination objects |
| Need custom jsui drawing | max-js-agent | Canvas size, drawing requirements, interaction behavior |
| Layout needs responsive behavior | max-patch-agent | Resize requirements and constraint rules |

## Layout Rules

- Presentation mode positions (`presentation_rect`) are this agent's authority
- Patching mode positions (`patching_rect`) defer to lead agent or layout engine
- These are independent: a box can have different positions in each mode
- Standard object spacing: ~80-120px vertical, ~150-200px horizontal (patching mode)
- Presentation spacing: minimum 10px between controls, 20px between groups
